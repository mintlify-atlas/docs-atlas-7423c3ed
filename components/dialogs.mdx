## Dialog

A modal dialog component built on Radix UI Dialog primitive.

### Import

```javascript
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogFooter,
  DialogTitle,
  DialogDescription,
  DialogClose,
} from "@/components/ui/Dialog"
```

### Component Architecture

The Dialog component is composed of multiple parts:

- `Dialog` - Root component that manages open/closed state
- `DialogTrigger` - Button that opens the dialog
- `DialogContent` - The modal content (centered, with overlay)
- `DialogHeader` - Header section with title and description
- `DialogFooter` - Footer section for actions
- `DialogTitle` - Dialog title (required for accessibility)
- `DialogDescription` - Optional description text
- `DialogClose` - Close button (automatic X button included)
- `DialogOverlay` - Background overlay (auto-included in DialogContent)
- `DialogPortal` - Portal component (auto-included in DialogContent)

### Component Source (Key Parts)

```javascript
import * as React from "react"
import * as DialogPrimitive from "@radix-ui/react-dialog"
import { X } from "lucide-react"
import { cn } from "../../utils/cn";

const Dialog = DialogPrimitive.Root
const DialogTrigger = DialogPrimitive.Trigger
const DialogClose = DialogPrimitive.Close

const DialogContent = React.forwardRef(({ className, children, ...props }, ref) => (
  <DialogPortal>
    <DialogOverlay />
    <DialogPrimitive.Content
      ref={ref}
      className={cn(
        "fixed left-[50%] top-[50%] z-50 grid w-full max-w-lg translate-x-[-50%] translate-y-[-50%] gap-4 border bg-background p-6 shadow-lg duration-200 data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 data-[state=closed]:zoom-out-95 data-[state=open]:zoom-in-95 sm:rounded-lg",
        className
      )}
      {...props}
    >
      {children}
      <DialogPrimitive.Close className="absolute right-4 top-4 rounded-sm opacity-70 hover:opacity-100">
        <X className="h-4 w-4" />
      </DialogPrimitive.Close>
    </DialogPrimitive.Content>
  </DialogPortal>
))
```

### Features

- **Portal rendering** - Dialog renders at document root to avoid z-index issues
- **Focus management** - Focus trapped within dialog when open
- **Keyboard support** - Escape key closes dialog
- **Scroll lock** - Body scroll locked when dialog is open
- **Animations** - Smooth fade and zoom animations
- **Overlay** - Semi-transparent black backdrop
- **Close button** - Automatic X button in top-right corner
- **Accessibility** - Full ARIA support via Radix UI

### Props

#### Dialog (Root)

- `open` - Controlled open state
- `onOpenChange` - Callback when open state changes
- `defaultOpen` - Initial open state
- `modal` - Whether to render as modal (default: true)

#### DialogContent

- `className` - Additional CSS classes
- `children` - Dialog content
- All standard div attributes

## Sheet

A slide-out panel component (drawer) built on Radix UI Dialog primitive.

### Import

```javascript
import {
  Sheet,
  SheetTrigger,
  SheetContent,
  SheetHeader,
  SheetFooter,
  SheetTitle,
  SheetDescription,
  SheetClose,
} from "@/components/ui/Sheet"
```

### Component Architecture

Sheet uses the same Radix Dialog primitive but with different styling:

- `Sheet` - Root component (SheetPrimitive.Root)
- `SheetTrigger` - Button that opens the sheet
- `SheetContent` - The slide-out panel content
- `SheetHeader` - Header section
- `SheetFooter` - Footer section  
- `SheetTitle` - Sheet title
- `SheetDescription` - Optional description
- `SheetClose` - Close button

### Variants

Sheet supports four side positions using class-variance-authority:

```javascript
const sheetVariants = cva(
  "fixed z-50 gap-4 bg-background p-6 shadow-lg transition ease-in-out",
  {
    variants: {
      side: {
        top: "inset-x-0 top-0 border-b data-[state=closed]:slide-out-to-top data-[state=open]:slide-in-from-top",
        bottom: "inset-x-0 bottom-0 border-t data-[state=closed]:slide-out-to-bottom data-[state=open]:slide-in-from-bottom",
        left: "inset-y-0 left-0 h-full w-3/4 border-r data-[state=closed]:slide-out-to-left data-[state=open]:slide-in-from-left sm:max-w-sm",
        right: "inset-y-0 right-0 h-full w-3/4 border-l data-[state=closed]:slide-out-to-right data-[state=open]:slide-in-from-right sm:max-w-sm",
      },
    },
    defaultVariants: {
      side: "right",
    },
  }
)
```

### Features

- **Four sides** - Slide from top, bottom, left, or right
- **Portal rendering** - Renders at document root
- **Focus management** - Focus trapped within sheet
- **Keyboard support** - Escape key closes sheet
- **Scroll lock** - Body scroll locked when open
- **Animations** - Smooth slide animations with different durations
- **Overlay** - Semi-transparent backdrop
- **Responsive** - Mobile-friendly widths
- **Close button** - Automatic X button

### Props

#### Sheet (Root)

- `open` - Controlled open state
- `onOpenChange` - Callback when open state changes  
- `defaultOpen` - Initial open state

#### SheetContent

- `side` - Panel position: `"top"`, `"right"`, `"bottom"`, `"left"` (default: `"right"`)
- `className` - Additional CSS classes
- `children` - Sheet content

## Complete Example

Here's a complete example using both Dialog and Sheet:

```jsx
import { Button } from "@/components/ui/Button"
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogDescription,
  DialogFooter,
  DialogClose,
} from "@/components/ui/Dialog"
import {
  Sheet,
  SheetTrigger,
  SheetContent,
  SheetHeader,
  SheetTitle,
} from "@/components/ui/Sheet"
import { Label } from "@/components/ui/Label"
import { Input } from "@/components/ui/Input"

function TransactionActions() {
  const [deleteOpen, setDeleteOpen] = useState(false)

  return (
    <div className="flex gap-2">
      {/* Edit transaction in a sheet */}
      <Sheet>
        <SheetTrigger asChild>
          <Button variant="outline">Edit</Button>
        </SheetTrigger>
        <SheetContent>
          <SheetHeader>
            <SheetTitle>Edit Transaction</SheetTitle>
          </SheetHeader>
          <div className="space-y-4 py-4">
            <div className="space-y-2">
              <Label htmlFor="amount">Amount</Label>
              <Input id="amount" type="number" placeholder="0.00" />
            </div>
            <div className="space-y-2">
              <Label htmlFor="description">Description</Label>
              <Input id="description" placeholder="Transaction description" />
            </div>
          </div>
          <div className="flex gap-2">
            <Button className="flex-1">Save Changes</Button>
          </div>
        </SheetContent>
      </Sheet>

      {/* Delete confirmation in a dialog */}
      <Dialog open={deleteOpen} onOpenChange={setDeleteOpen}>
        <DialogTrigger asChild>
          <Button variant="destructive">Delete</Button>
        </DialogTrigger>
        <DialogContent>
          <DialogHeader>
            <DialogTitle>Delete Transaction</DialogTitle>
            <DialogDescription>
              Are you sure you want to delete this transaction? This action cannot be undone.
            </DialogDescription>
          </DialogHeader>
          <DialogFooter>
            <DialogClose asChild>
              <Button variant="outline">Cancel</Button>
            </DialogClose>
            <Button 
              variant="destructive"
              onClick={() => {
                // Delete transaction
                setDeleteOpen(false)
              }}
            >
              Delete
            </Button>
          </DialogFooter>
        </DialogContent>
      </Dialog>
    </div>
  )
}
```

## When to Use

### Use Dialog when:

- Requiring immediate user attention
- Confirming destructive actions
- Displaying critical information
- Form is short and focused
- Action affects entire application

### Use Sheet when:

- Displaying related or contextual information
- Longer forms or multi-step workflows
- Navigation menus on mobile
- Filters or settings panels
- Action is contextual to specific content
