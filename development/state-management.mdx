TradeMaster Transactions uses Redux Toolkit for centralized state management with redux-persist for state persistence across sessions.

## Store Configuration

The Redux store is configured in `src/store/Store.js` with the following features:

- **Redux Toolkit** for simplified Redux setup
- **Redux Persist** for localStorage persistence
- **Encryption support** (currently disabled but available)
- **Logout handling** that clears sensitive state

### Store Setup

From `src/store/Store.js:59-69`:

```javascript
export const store = configureStore({
  reducer: persistedReducer,
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      serializableCheck: {
        ignoredActions: [FLUSH, REHYDRATE, PAUSE, PERSIST, PURGE, REGISTER],
      },
    }),
});

export const persistor = persistStore(store);
```

### Persistence Configuration

From `src/store/Store.js:25-37`:

```javascript
const persistConfig = {
  key: "root",
  storage,
  whitelist: ["auth", "loading", "setup", "events", "configData", "emailStatus"],
  transforms: [
    // encryptTransform({
    //   secretKey: 'default_key_test',
    //   onError: (error) => {
    //     console.error('Encriptación fallida:', error);
    //   },
    // }),
  ],

```

**Persisted Slices**:
- `auth` - User authentication state
- `loading` - Global loading indicators
- `setup` - Application setup data
- `events` - Event listings and details
- `configData` - Configuration from Firebase
- `emailStatus` - Email send tracking

<Note>
  Encryption is available via `redux-persist-transform-encrypt` but currently commented out. Uncomment and configure a secure secret key for production to encrypt sensitive data in localStorage.
</Note>

## Redux Slices

The application state is divided into feature-based slices located in `src/store/apps/`.

### Auth Slice

**Location**: `src/store/apps/auth/authSlice.js`

**Purpose**: Manages user authentication state

**State Shape**:
```javascript
{
  isAuthenticated: false,
  isInitialized: false,
  initialEmail: null,
  userToComplete: null,
  user: null
}
```

**Actions**:

<CodeGroup>
```javascript authStateChanged

// Update authentication state when user logs in/out
dispatch(authStateChanged({
  isAuthenticated: true,
  user: {
    uid: '123',
    email: 'user@example.com',
    displayName: 'John Doe',
    date: {
      created: timestamp,
      updated: timestamp,
      validated: timestamp
    }
  }
}));
```

```javascript setInitialEmail
import { setInitialEmail } from '@/store/apps/auth/authSlice';

// Store email during registration flow
dispatch(setInitialEmail({ email: 'user@example.com' }));
```

```javascript setUserToComplete
import { setUserToComplete } from '@/store/apps/auth/authSlice';

// Store partial user data during registration
```

</CodeGroup>

**Usage Example**:

### Email Status Slice

**Location**: `src/store/apps/emailStatus/emailStatusSlice.js`

**Purpose**: Tracks which order confirmation emails have been sent to prevent duplicates

**State Shape**:
```javascript
{
  sentEmails: {
    'order_id_1': true,
    'order_id_2': true
  }
}
```

**Actions**:

<CodeGroup>
```javascript setEmailSent
import { setEmailSent } from '@/store/apps/emailStatus/emailStatusSlice';

// Mark email as sent for an order
dispatch(setEmailSent({ orderId: 'order_123' }));
```

```javascript clearEmailStatus
import { clearEmailStatus } from '@/store/apps/emailStatus/emailStatusSlice';

// Clear all email status (e.g., on logout)
dispatch(clearEmailStatus());
```
</CodeGroup>

**Usage Example**:
```javascript
import { useSelector, useDispatch } from 'react-redux';
import { setEmailSent } from '@/store/apps/emailStatus/emailStatusSlice';

const OrderConfirmation = ({ orderId }) => {
  const sentEmails = useSelector((state) => state.emailStatus.sentEmails);
  const dispatch = useDispatch();

  const sendConfirmationEmail = async () => {
    if (!sentEmails[orderId]) {
      await sendEmail(orderId);
      dispatch(setEmailSent({ orderId }));
    }
  };

  return <button onClick={sendConfirmationEmail}>Send Email</button>;
};
```

## Root Reducer and Logout

The root reducer includes special handling for logout (`src/store/Store.js:48-55`):

```javascript
const rootReducer = (state, action) => {
  if (action.type === LOGOUT) {
    storage.removeItem('persist:root');
    const { setup, events, configData } = state || {};
    state = { setup, events, configData };
  }
  return appReducer(state, action);
};
```

**Logout Action** (`src/store/Store.js:71`):

```javascript
import { logout } from '@/store/Store';
import { useDispatch } from 'react-redux';

const LogoutButton = () => {
  const dispatch = useDispatch();

  const handleLogout = () => {
    dispatch(logout());
    // This will:
    // 1. Clear persisted state from localStorage
    // 2. Reset auth and user data
    // 3. Preserve setup, events, and configData
  };

  return <button onClick={handleLogout}>Logout</button>;
};
```

<Note>
  On logout, the store preserves `setup`, `events`, and `configData` so these don't need to be re-fetched. Only user-specific state (`auth`, `loading`, `emailStatus`) is cleared.
</Note>

## Best Practices

<Note>
**State Organization**
- Keep slices focused on a single domain
- Use normalized state shape for entities
- Avoid deeply nested state structures
</Note>

<Note>
**Performance**
- Use memoized selectors for derived state (consider `reselect`)
- Select only the data you need in components
- Avoid selecting large objects unnecessarily
</Note>

<Note>
**Async Operations**
- Use Redux Toolkit's `createAsyncThunk` for async logic
- Handle loading, success, and error states
- Show loading indicators during async operations
</Note>

<Note>
**Security**
- Enable encryption for sensitive data in production
- Never store passwords or tokens in Redux
- Use secure secret keys for encryption
- Clear sensitive state on logout
</Note>

## Common Patterns

### Fetching Data on Mount

```javascript
import { useEffect } from 'react';
import { useDispatch, useSelector } from 'react-redux';
import { fetchSetup } from '@/store/apps/setup/configDataSlice';

const App = () => {
  const dispatch = useDispatch();
  const configData = useSelector((state) => state.configData.config_data);

  useEffect(() => {
    if (Object.keys(configData).length === 0) {
      dispatch(fetchSetup());
    }
  }, [dispatch, configData]);

  return <div>App Content</div>;
};
```

### Loading State Pattern

```javascript
import { useDispatch } from 'react-redux';
import { setLoadingData } from '@/store/apps/loading/loadingSlice';

const fetchData = async () => {
  dispatch(setLoadingData({ loading: true }));
  
  try {
    const data = await api.fetchData();
    dispatch(setData(data));
  } catch (error) {
    console.error(error);
  } finally {
    dispatch(setLoadingData({ loading: false }));
  }
};
```

### Conditional Rendering Based on Auth

```javascript
import { useSelector } from 'react-redux';

const ProfileButton = () => {
  const isAuthenticated = useSelector((state) => state.auth.isAuthenticated);
  const user = useSelector((state) => state.auth.user);

  if (!isAuthenticated) {
    return <Link to="/auth/login">Login</Link>;
  }

  return (
    <DropdownMenu>
      <DropdownMenuTrigger>{user.displayName}</DropdownMenuTrigger>
      <DropdownMenuContent>
        <DropdownMenuItem>Profile</DropdownMenuItem>
        <DropdownMenuItem>Logout</DropdownMenuItem>
      </DropdownMenuContent>
    </DropdownMenu>
  );
};
```
