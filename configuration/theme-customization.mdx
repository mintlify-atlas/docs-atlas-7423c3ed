## Overview

TradeMaster Transactions Portales provides extensive theming capabilities through Tailwind CSS, CSS variables, and a dynamic theme provider. You can customize colors, spacing, typography, and portal-specific branding.

## Theme Architecture

The theme system consists of three main components:

1. **Tailwind Configuration** - Base design system tokens
2. **CSS Variables** - Runtime color customization
3. **Theme Provider** - Dynamic portal-specific themes

### Color Format (HSL)

All colors use HSL (Hue, Saturation, Lightness) format:

- **Hue**: 0-360 (color wheel position)
- **Saturation**: 0-100% (color intensity)
- **Lightness**: 0-100% (brightness)

**Example:**
```css
--primary: 220 70% 50%; /* Blue color */
```

Use with `hsl()` function:
```css
color: hsl(var(--primary));
```

## Dynamic Theme Provider

The `ThemeProvider` component enables portal-specific color customization at runtime:

```jsx src/components/ThemeProvider.jsx
import { useEffect } from 'react';
import { useSelector } from 'react-redux';

const ThemeProvider = ({ children }) => {
  const setup = useSelector((state) => state.setup.Setup.metadata);

  useEffect(() => {
    if (setup?.color_primary && setup?.color_secondary) {
      // Update CSS custom properties
      document.documentElement.style.setProperty('--color-primary', setup.color_primary);
      document.documentElement.style.setProperty('--color-secondary', setup.color_secondary);
      
      // Convert HEX to HSL for Tailwind
      const primaryHSL = hexToHSL(setup.color_primary);
      const secondaryHSL = hexToHSL(setup.color_secondary);
      
      document.documentElement.style.setProperty('--primary', primaryHSL);
      document.documentElement.style.setProperty('--secondary', secondaryHSL);
      
      // Update foreground colors
      document.documentElement.style.setProperty('--primary-foreground', '255 255 255');
      document.documentElement.style.setProperty('--secondary-foreground', '255 255 255');
    }
  }, [setup?.color_primary, setup?.color_secondary]);

  return children;
};

// Convert HEX to HSL
function hexToHSL(hex) {
  hex = hex.replace('#', '');
  
  const r = parseInt(hex.substring(0, 2), 16) / 255;
  const g = parseInt(hex.substring(2, 4), 16) / 255;
  const b = parseInt(hex.substring(4, 6), 16) / 255;

  const max = Math.max(r, g, b);
  const min = Math.min(r, g, b);
  let h, s, l = (max + min) / 2;

  if (max === min) {
    h = s = 0;
  } else {
    const d = max - min;
    s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
    
    switch (max) {
      case r: h = (g - b) / d + (g < b ? 6 : 0); break;
      case g: h = (b - r) / d + 2; break;
      case b: h = (r - g) / d + 4; break;
    }
    
    h /= 6;
  }

  h = Math.round(h * 360);
  s = Math.round(s * 100);
  l = Math.round(l * 100);

  return `${h} ${s}% ${l}%`;
}

export default ThemeProvider;
```

### How It Works

<Steps>
  <Step title="Portal Configuration Loaded">
    When a portal is accessed, its configuration (including `color_primary` and `color_secondary`) is loaded from Redux state.
  </Step>

  <Step title="Colors Applied Dynamically">
    The `ThemeProvider` watches for color changes and updates CSS variables in real-time.
  </Step>

  <Step title="Automatic Conversion">
    HEX colors from the database are automatically converted to HSL format for Tailwind compatibility.
  </Step>

  <Step title="UI Updates Instantly">
    All components using `primary` or `secondary` colors update without page refresh.
  </Step>

## Customizing Colors

### Option 1: Update Portal Configuration

The easiest way to change colors is through the portal configuration in Firestore:

```javascript
import { db } from './guards/firebase/firebase';
import { doc, updateDoc } from 'firebase/firestore';

const updatePortalColors = async (portalId, colors) => {
  const portalRef = doc(db, 'portals', portalId);
  await updateDoc(portalRef, {
    'metadata.color_primary': colors.primary,
    'metadata.color_secondary': colors.secondary
  });
};

// Usage
await updatePortalColors('portal-123', {
  primary: '#3B82F6',   // Blue
  secondary: '#10B981'  // Green
});
```

### Option 2: Modify CSS Variables

For global changes, edit `src/index.css`:

```css
:root {
  /* Change these values */
  --primary: 220 70% 50%;        /* Blue */
  --secondary: 142 76% 36%;      /* Green */
  --accent: 24 95% 53%;          /* Orange */
}
```

### Option 3: Override in Tailwind Config

Add custom colors to `tailwind.config.js`:

```javascript
theme: {
  extend: {
    colors: {
      brand: {
        primary: '#3B82F6',
        secondary: '#10B981',
        accent: '#F59E0B'
      }
    }
  }
}
```

Use in components:
```jsx
<button className="bg-brand-primary text-white">
  Click me
</button>
```

## Dark Mode Implementation

<Steps>
  <Step title="Create Dark Mode Toggle">
    ```jsx
    import { useState, useEffect } from 'react';
    
    const DarkModeToggle = () => {
      const [isDark, setIsDark] = useState(false);
      
      useEffect(() => {
        const isDarkMode = document.documentElement.classList.contains('dark');
        setIsDark(isDarkMode);
      }, []);
      
      const toggleDarkMode = () => {
        document.documentElement.classList.toggle('dark');
        setIsDark(!isDark);
        localStorage.setItem('darkMode', !isDark);
      };
      
      return (
        <button onClick={toggleDarkMode}>
          {isDark ? '🌞 Light' : '🌙 Dark'}
        </button>
      );
    };
    ```
  </Step>

  <Step title="Persist Dark Mode Preference">
    ```javascript
    // In your root component or App.jsx
    useEffect(() => {
      const darkMode = localStorage.getItem('darkMode') === 'true';
      if (darkMode) {
        document.documentElement.classList.add('dark');
      }
    }, []);
    ```
  </Step>

  <Step title="Use Dark Mode Classes">
    ```jsx
    <div className="bg-white dark:bg-gray-900 text-black dark:text-white">
      Content adapts to theme
    </div>
    ```
  </Step>
</Steps>

## Troubleshooting

### Colors Not Updating

**Problem**: Changed CSS variables but colors remain the same

**Solution**: 
1. Clear browser cache
2. Rebuild the application: `npm run build`
3. Check browser DevTools to verify CSS variables are applied

### Theme Provider Not Working

**Problem**: Portal colors not applying dynamically

**Solution**:
1. Verify `ThemeProvider` wraps your app in the component tree
2. Check Redux state contains `color_primary` and `color_secondary`
3. Ensure portal configuration is loaded before rendering

### Dark Mode Not Toggling

**Problem**: Dark mode toggle doesn't change appearance

**Solution**:
1. Verify `darkMode: ["class"]` is set in `tailwind.config.js`
2. Check that `dark:` variants are used in your components
3. Ensure CSS variables are defined for both `:root` and `.dark`
