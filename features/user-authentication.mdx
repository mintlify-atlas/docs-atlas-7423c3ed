## Overview

TradeMaster uses Firebase Authentication for secure user management with email/password authentication, protected routes, and automatic session management. The system implements authentication guards to control access to different parts of the application.

### Login

Existing users sign in with email and password:

```javascript title="src/views/auth/login/Login.js:44-108"
onSubmit: async (values, { setStatus, setSubmitting }) => {
  try {
    setSubmit(true);
    
    // Step 1: Sign in with Firebase
    const userCredential = await signInWithEmailAndPassword(
      auth,
      values.email,
      values.password
    );

    const userUid = userCredential.user.uid;

    // Step 2: Fetch user data from Firestore
    const querySnapshot = await getDoc(doc(db, "u_users", userUid));

    // Step 3: Check if user is active
    if (querySnapshot.data()?.status !== true) {
      await signOut(auth);
      setSubmit(false);
      
      dispatch(authStateChanged({
        isAuthenticated: false,
        user: null,
      }));
    } else {
      // Step 4: Update Redux state with user data
      dispatch(authStateChanged({
        isAuthenticated: true,
        user: {
          id: userUid,
          email: querySnapshot.data()?.email,
          name: querySnapshot.data()?.name,
          ...querySnapshot.data(),
        },
      }));
      
      dispatch(setInitialEmail({
        email: null,
      }));
    }
  } catch (error) {
    toast.error(getErrorMessage(error.code), {
      position: "bottom-center",
      autoClose: 7000,
      theme: "light",
    });
    setSubmitting(false);
    setSubmit(false);
  }
}
```

## Authentication Guards

Route protection is implemented using guard components:

### AuthGuard (Protected Routes)

Ensures only authenticated users can access protected routes:

```javascript title="src/guards/authGuard/AuthGuard.js"
import { useEffect } from 'react';
import { useSelector } from 'react-redux';
import { useNavigate, useLocation } from 'react-router-dom';

const AuthGuard = ({ children }) => {
  const isAuthenticated = useSelector((state) => state.auth.isAuthenticated);
  const navigate = useNavigate();
  const location = useLocation();

  useEffect(() => {
    if (!isAuthenticated) {
      // Save current URL to return after login
      const currentUrl = location.pathname + location.search;
      sessionStorage.setItem('returnUrl', currentUrl);
      navigate('/auth/checkEmail', { replace: true });
    }
  }, [isAuthenticated, location, navigate]);

  return children;
};

export default AuthGuard;
```

### GuestGuard (Auth Pages)

Redirects authenticated users away from login/register pages:

```javascript title="src/guards/authGuard/GuestGuard.js"
import { useEffect } from 'react';
import { useSelector } from 'react-redux';
import { useNavigate } from 'react-router-dom';

const GuestGuard = ({ children }) => {
  const isAuthenticated = useSelector((state) => state.auth.isAuthenticated);
  const navigate = useNavigate();

  useEffect(() => {
    if (isAuthenticated) {
      // Read the returnUrl from sessionStorage
      const returnUrl = sessionStorage.getItem('returnUrl');

      // Redirect to saved URL or home
      const destination = returnUrl || '/';

      // Clear the returnUrl
      if (returnUrl) {
        sessionStorage.removeItem('returnUrl');
      }

      navigate(destination, { replace: true });
    }
  }, [isAuthenticated, navigate]);

  return children;
};

export default GuestGuard;
```

## State Management

Authentication state is managed in Redux:

```javascript title="Redux Auth Slice"
import { createSlice } from "@reduxjs/toolkit";

const initialState = {
  isAuthenticated: false,
  user: null,
  initialEmail: null,
  userToComplete: null,
};

export const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    authStateChanged: (state, action) => {
      state.isAuthenticated = action.payload.isAuthenticated;
      state.user = action.payload.user;
    },
    setInitialEmail: (state, action) => {
      state.initialEmail = action.payload.email;
    },
    setUserToComplete: (state, action) => {
      state.userToComplete = action.payload.user;
    },
  },
});

export const {
  authStateChanged,
  setInitialEmail,
  setUserToComplete
} = authSlice.actions;

export default authSlice.reducer;
```

### Return URL Functionality

When unauthenticated users try to access protected routes:

1. The current URL is saved to `sessionStorage`
2. User is redirected to login
3. After successful login, user is redirected back to the original URL


## Sign Out

Implement sign out functionality:

```javascript title="Sign Out"
import { signOut } from 'firebase/auth';
import { auth } from './guards/firebase/firebase';
import { useDispatch } from 'react-redux';
import { authStateChanged } from './store/apps/auth/authSlice';

const handleSignOut = async () => {
  try {
    await signOut(auth);

      isAuthenticated: false,
      user: null,
    }));
```

## Best Practices

1. **Always validate user status** - Check if `status === true` before allowing access
2. **Store minimal data in Redux** - Keep only necessary user info in state
3. **Use Firebase security rules** - Protect Firestore data with proper rules
4. **Implement proper error handling** - Show user-friendly messages for auth errors
5. **Save return URLs** - Allow users to continue where they left off after login
6. **Validate email addresses** - Ensure users verify their email for account security
7. **Use HTTPS only** - Never transmit authentication data over unencrypted connections
8. **Clear sensitive data on logout** - Remove all user data from state and storage
9. **Implement rate limiting** - Protect against brute force attacks
10. **Log authentication events** - Track logins, logouts, and failed attempts for security
