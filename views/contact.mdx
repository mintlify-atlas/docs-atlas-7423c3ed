## Overview

The Contact view (`src/views/contact/Contact.js`) provides a comprehensive contact interface for users to get in touch with the event organization. It combines a contact form with company information, social media links, location map, and FAQs.

## Route Information

**Path**: `/contacto`  
**Layout**: `FullLayout`  
**Authentication**: Public (no auth required)  
**Guard**: None

From `src/routes/router.js:37`:

```javascript
{ path: "/contacto", element: <Contact /> }
```

## Component Structure

The Contact view includes several sections:

### 1. Hero Banner

A gradient banner with the page title and description.

### 2. Contact Information Card

Displays company contact details from the portal's metadata:
- Phone number with schedule
- Email addresses
- Office address
- Social media links (Facebook, Instagram, Twitter/X, YouTube)

### 3. Contact Form

A fully validated form with the following fields:
- **Name** (required)
- **Email** (required with format validation)
- **Phone** (optional)
- **Subject** (required)
- **Message** (required, textarea)

### 4. Location Map

Embedded Google Maps iframe showing the organization's location (if coordinates are configured).

### 5. FAQs Section

Displays frequently asked questions configured in the portal metadata.

### Validation Logic

From `src/views/contact/Contact.js:41-64`:

```javascript
const validateForm = () => {
  const newErrors = {};

  if (!formData.name.trim()) {
    newErrors.name = 'El nombre es requerido';
  }

  if (!formData.email.trim()) {
    newErrors.email = 'El correo electrónico es requerido';
  } else if (!/\S+@\S+\.\S+/.test(formData.email)) {
    newErrors.email = 'El correo electrónico no es válido';
  }

  if (!formData.subject.trim()) {
    newErrors.subject = 'El asunto es requerido';
  }

  if (!formData.message.trim()) {
    newErrors.message = 'El mensaje es requerido';
  }

  setErrors(newErrors);
  return Object.keys(newErrors).length === 0;
};
```

### Form Submission

From `src/views/contact/Contact.js:66-96`:

```javascript
const handleSubmit = (e) => {
  e.preventDefault();

  if (validateForm()) {
    dispatch(setLoadingData({ loading: true }));

    // Simulates form submission
    setTimeout(() => {
      dispatch(setLoadingData({ loading: false }));
      toast.success("Mensaje enviado correctamente. Nos pondremos en contacto contigo pronto.", {
        position: "bottom-center",
        autoClose: 5000,
        hideProgressBar: false,
        closeOnClick: true,
        pauseOnHover: true,
        draggable: true,
        progress: undefined,
        theme: "light",
      });

      // Reset form
      setFormData({
        name: '',
        email: '',
        phone: '',
        subject: '',
        message: ''
      });
    }, 1500);
  }
};
```

<Note>
  The current implementation simulates email sending. In a production environment, this should be replaced with a Firebase Cloud Function call to `send_email` or an email service integration.
</Note>

## Enhancement Opportunities

<Warning>
  The contact form currently uses a simulated submission (setTimeout). For production use, implement actual email sending via:
</Warning>

1. **Firebase Cloud Function Integration**

```javascript
import { send_email } from '../../guards/firebase/firebase';

const handleSubmit = async (e) => {
  e.preventDefault();
  
  if (validateForm()) {
    dispatch(setLoadingData({ loading: true }));
    
    try {
      await send_email({
        to: setup.metadata.contact_info.emails[0],
        subject: `Contact Form: ${formData.subject}`,
        message: `
          Name: ${formData.name}
          Email: ${formData.email}
          Phone: ${formData.phone || 'N/A'}
          
          Message:
          ${formData.message}
        `
      });
      
      toast.success("Mensaje enviado correctamente.");
      setFormData({ name: '', email: '', phone: '', subject: '', message: '' });
    } catch (error) {
      toast.error("Error al enviar el mensaje. Intenta nuevamente.");
    } finally {
      dispatch(setLoadingData({ loading: false }));
    }
  }
};
```

2. **Email Service Integration** - SendGrid, Mailgun, or AWS SES
3. **CRM Integration** - Store inquiries in Firestore for tracking
